Project information

  • Category: JS API and ArcGIS Online Mapping
  • Client: Halton Conservation Authority
  • Project date: 01 March, 2020

Image Descriptions

Image 1) This operations dashboard uses pre-processed data from a web map created in AGOL. The summarize within tool was used to find areas in square meters of building polygons that are within the flood buffer zones for 3 buffer distances: 10m, 30m, 50m. The Web Map displays the areas of risk associated with the total area of house polygons (in Sq. Meters) within a stretch of the river segment. This allows users to easily see what areas have a higher risk for a 10m, 30m, or 50m flood buffer around the river. Sub watersheds where symbolized to show each of the different watershed zones. This is for environmentalists or the conservation authority to see what watersheds have more risk of flood damage to buildings. When developing the Pie chart, I configured the category field to display the 7 largest Watersheds, opposed to all for simplicity and clarity. I also added functionality to the pie chart so users can click a pie segment and it will zoom to the watershed selected, flash, and display it on the map. It also shows a popup of information related to that area of watershed. Unclicking the pie segment brings the map back to the default extent.

Image 2) Initially JavaScript 3.31 was used to compare with 4.14. It was determined that JS 4.14 had far better functionality and appeared less restrictive for coding in widgets such as scale bar, search bar, and on click address locator. The dashboard created with 3.31 is a simple interface that allows the user to click directly on the flood zones and watersheds to retrieve information about them. A div was used to segment the header, map area and legend.

Image 3) JavaScript API 4.14 was found to be much easier to code and had helpful JavaScript tutorials. Firstly, a code was developed that inserted a legend with a scroll bar and a layers list that allows the user to toggle on and off the layers they want to see. Next, all the new widgets were inserted to the required and function area of the code for it to display properly. After this a popup widget was inserted to show the longitude and latitude, as well as the scale of the map which changes automatically as the cursor moves over the map. A search section was also created to allow the user to search their address and see if it is in a flood zone by referencing the Flood Risk Layer. Finally, the most useful part of the app was the onclick address locator coded so users can click an area anywhere in the flood area and receive the address and postal code of that area.
